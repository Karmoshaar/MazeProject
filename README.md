مشروع توليد وحل المتاهات (Maze Generator & Solver)
يهدف هذا المشروع إلى تصميم وتنفيذ نظام متكامل بلغة Python لتوليد متاهات عشوائية وحلها باستخدام مجموعة من خوارزميات البحث، مع إجراء تحليل مقارن لأداء هذه الخوارزميات من حيث زمن التنفيذ، استهلاك الذاكرة، وطول المسار الناتج.
أهداف المشروع
فهم آلية توليد المتاهات باستخدام الخوارزميات التراجعية.
تطبيق خوارزميات البحث المختلفة على مشكلة إيجاد المسار.
مقارنة أداء الخوارزميات من ناحية الكفاءة.
تمثيل المتاهة والمسارات الناتجة بشكل مرئي.
الميزات الأساسية
1. توليد المتاهات
يتم توليد المتاهة باستخدام خوارزمية Recursive Backtracking، والتي تنتج متاهة مثالية لا تحتوي على حلقات وتضمن وجود مسار وحيد بين أي نقطتين داخل المتاهة.
2. خوارزميات الحل
يدعم المشروع أربع خوارزميات للبحث عن المسار:
البحث بالعمق أولاً (DFS)
البحث بالعرض أولاً (BFS)
خوارزمية ديكسترا (Dijkstra)
خوارزمية A* (A-Star)
3. تحليل الأداء
يتم قياس أداء كل خوارزمية اعتماداً على:
زمن التنفيذ (بالميلي ثانية)
استهلاك الذاكرة (بالكيلوبايت)
عدد الخطوات في المسار النهائي
وذلك باستخدام مكتبات Python القياسية مثل time و tracemalloc.
4. التصور البياني
يتم رسم المتاهة والمسارات الناتجة باستخدام مكتبة Matplotlib، مما يساعد على فهم طريقة عمل كل خوارزمية ومقارنة نتائجها بصرياً.
هيكلية المشروع
تم تقسيم المشروع إلى وحدات برمجية منظمة كما يلي:
maze/cell.py
يمثل الخلية الأساسية في المتاهة، ويحتوي على حالة الجدران الأربعة والعلاقات مع الخلايا المجاورة.
maze/maze.py
يدير الشبكة ثنائية الأبعاد للمتاهة ويتحكم بآلية الربط بين الخلايا.
maze/generator.py
مسؤول عن توليد المتاهة باستخدام المكدس (Stack) وخوارزمية Recursive Backtracking.
maze/solver.py
يحتوي على تنفيذ خوارزميات البحث المختلفة وإعادة بناء المسار الناتج.
analysis/metrics.py
مخصص لقياس زمن التنفيذ واستهلاك الذاكرة لكل خوارزمية.
visualization/draw.py
مسؤول عن تمثيل المتاهة والمسارات بيانياً.
آلية العمل
يعمل البرنامج وفق الخطوات التالية:
إنشاء شبكة المتاهة بالأبعاد المطلوبة.
توليد المتاهة بشكل عشوائي.
ربط الخلايا المجاورة لتحديد المسارات المتاحة.
تشغيل خوارزميات البحث الأربع وقياس أدائها.
عرض جدول مقارنة للنتائج ورسم المتاهة مع المسار الأقصر الناتج عن خوارزمية A*.
الخوارزميات المستخدمة
DFS (Depth-First Search)
يستكشف المسارات بعمق قبل الرجوع للخلف، وقد لا يعطي دائماً أقصر مسار.
BFS (Breadth-First Search)
يستكشف الخلايا طبقة بطبقة ويضمن إيجاد أقصر مسار في المتاهات غير الموزونة.
Dijkstra
يبحث عن المسار الأقل كلفة، ويُعد مناسباً عند وجود أوزان مختلفة للمسارات.
A*
يستخدم دالة تقديرية (Heuristic) لتوجيه البحث نحو الهدف بكفاءة أعلى.

طريقة التشغيل
لتشغيل البرنامج يتم تنفيذ الملف الرئيسي:

في نهاية التنفيذ، يعرض البرنامج جدولاً يقارن بين أداء الخوارزميات، كما يتم رسم المتاهة والمسار الأفضل.
